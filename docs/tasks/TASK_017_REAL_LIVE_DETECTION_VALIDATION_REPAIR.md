## 真机失败结论

## 问题分类

### A. 当前轮次必须立即修复的 bug

- 正式用户训练路径仍显示“模拟识别”或 `+1 / -1`
  - 判定：违反已冻结的正式路径边界
  - 当前轮次必须修复
- 正常深蹲未能驱动 `live` 计数
  - 判定：阻塞“真实可用闭环”
  - 当前轮次必须修复
- 校准完成 / 失败反馈不明确
  - 判定：阻塞用户理解与真机验证复现
  - 当前轮次必须修复

### B. 当前轮次必须修订的验收口径

- 校准流程不能只依赖一次深蹲或过短窗口
  - 新口径：首次校准应支持约 `10` 次左右深蹲或等价的充分采样窗口
  - 验收必须记录校准开始、进行中、完成 / 失败反馈
- 校准复用口径需要明确
  - 新口径：首次安装 / 首次无校准结果时执行校准，后续训练默认复用已保存校准结果
  - 若校准失败或中断，不能进入正式训练，只能重试或退出
- 防误触口径需要后置到基础识别跑通后判断
  - 新口径：半蹲、抬腕、轻微晃动若出现误计，应先记录为调优项；是否阻塞由总控按误计频率和影响范围判断

### C. 后续新增范围，不并入当前 Watch 修复任务

- iPhone 端“重置校准”入口
  - 判定：涉及 iPhone Companion 页面与跨端配置/存储口径
  - 不应顺手混入当前 Watch 端修复
  - 后续需单独补任务契约后执行
- 高准确率统计 KPI
  - 判定：需要多轮真机样本、调优记录与验收矩阵
  - 当前只保留“真实可用闭环”口径，不宣称高准确率稳定
- 多用户 / 多佩戴方式调优矩阵
  - 判定：属于后续调优阶段，不阻塞当前 bug 修复启动

## 后续修复任务拆分

### 修复任务 1：正式路径 UI 清理

- 移除正式用户可见的“模拟识别”入口
- 移除正式用户可见的 `+1 / -1`
- 内部调试能力只能通过 Debug / Beta / 隐藏入口保留，不得在正式训练路径出现

### 修复任务 2：校准体验与采样策略修复

- 增加明确校准引导
- 增加校准进行中提示
- 增加校准完成 / 失败提示
- 将校准采样改为约 `10` 次深蹲或足够采样窗口
- 校准失败 / 中断后不进入训练

### 修复任务 3：真实识别链路修复

- 排查 `CoreMotion -> SquatMotionSample` 映射
- 排查阈值是否导致正常深蹲无法进入 `descending / bottom / ascending / repDetected`
- 至少让正常深蹲能驱动 `live` 计数

### 修复任务 4：真机验证回传更新

- 更新手动验证口径，明确校准采样期望与判定标准
- 重新执行真实设备验证后，回写通过 / 失败、影响范围与剩余调优项
