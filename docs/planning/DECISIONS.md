# 决策基线

## 已确认决策

- 产品形态：`iPhone + Watch companion`
- 训练主状态源：`Apple Watch`
- 首发版本定位：`1.0 可上架 MVP`
- 首发范围：不做账号、云同步、社交、AI 纠错
- `1.0` iPhone Companion 页面范围：首次引导、默认参数配置、帮助与支持、轻量基础信息入口
- `1.0` iPhone 信息架构：单个 companion 首页承接入口与说明，不使用 `Tab Bar`
- iPhone 不承接实时训练控制，也不承接实时训练状态镜像
- 历史页 / 训练结果页不纳入 `1.0`
- 基础设置不作为独立复杂页面，优先并入帮助与支持信息架构
- `6.7` 首批同步范围：`WorkoutConfig` 下发到 Watch，`WorkoutSummary` 回传到 iPhone；`WatchStateSnapshot` 仅保留为非首批用户可见预留
- `6.7` 首批可见承载方式：不新增历史页 / 结果页；若需要展示训练回传，仅允许在 companion 首页承接“最近一次训练摘要”单卡
- `6.7` 默认参数冲突规则：以 `SyncPayload.updatedAt` 最新值为准，但不改写已经启动的 Watch 当前训练会话
- `6.7` 重试边界：只允许轻量、有限、不可阻塞训练主流程的重试，不做独立同步中心或手动重试队列
- HealthKit 策略：`1.0` 目标写入 `Health app` 的 workout
- `6.8` 首批权限范围：仅限 Watch 侧最小 workout 相关 `HealthKit` 权限；`iPhone` 不单独请求 `HealthKit`，`1.0` 不读取心率、卡路里等额外健康数据
- `6.8` `Workout Session` / 写入边界：系统会话仅覆盖 Watch 端一次明确开始的训练主链路；训练完成路径最多写入一条 workout，中止 / 取消 / 拒权路径不写入，且不阻塞本地训练
- `6.9` 数据边界：`1.0` 首批只承接 `UserDefaults` 训练默认配置存储、`WatchConnectivity` 最小同步 payload、Watch 完成路径写入单条 `Health app` workout；不扩展账号数据与额外健康数据类型
- `6.5` 正式路径边界：正式用户默认使用 `live` 识别路径；模拟识别与 `+1 / -1` 仅保留内部调试可见入口；首次训练前执行一次性个人校准并持久化复用
- `6.5` 校准管理边界：无有效校准结果时不得进入正式训练；校准失败 / 中断后只允许重试或退出；iPhone 端“重置校准”入口属于后续新增范围，必须另补任务契约后实现
- 首发 KPI：下载量达到 `100`
- 首发市场策略：中国区、中文、免费

## 使用规则

- 决策变更必须先更新本文件
- 决策未更新前，旧规则继续生效
- 聊天中的临时表述不构成决策变更
