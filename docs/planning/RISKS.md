# 风险基线

## 当前风险

### 高风险

- 动作识别准确率不可控，影响核心体验

### 中风险

- `6.8` 已完成 Watch 侧最小 `HealthKit capability`、权限文案、拒权非阻塞降级、`Workout Session` 生命周期接线和完成路径写入 `Health app` 的代码落地，并已通过最小 `build` / `build-for-testing` 与真实配对设备上的首次授权、拒权降级、完成训练写入 `Health app` 手动验证；当前剩余主要风险是完整 `XCTest` / `test-without-building` 在执行阶段仍可能卡住，以及真实设备更大范围回归尚未覆盖
- `6.9` 数据与隐私合规已完成边界冻结；在免费 `Apple Personal Team` 路径下，`App Privacy` 问卷、`TestFlight`、提审后台动作不作为本轮 checkpoint 阻塞项。后续若切换到提审路径，仍需用户手动执行并回传，且平台填报与仓库口径不一致时将阻塞提审收口
- `6.9` 最小手动验证已在 `2026-04-19` 完成 1 次回传（本地安装运行、完成训练写入、倒计时取消不写入）；当前仍属于单次样本验证，后续实现或权限链路变化后需复测
- `6.7` 最小同步闭环已完成代码落地，并已补到 1 次真实配对设备闭环验证；但断连恢复、多轮往返同步与更大范围真机回归仍未覆盖，当前不能把跨设备链路视为“已稳定”
- `TASK_008` 已冻结 `6.6` 的实现文件范围与本地持久化边界；若后续实现阶段绕过 `PhoneCompanionViewModel` / `WorkoutConfigStoring` 直接在页面层操作存储或同步抽象，将导致 `6.6` 与 `6.7` 职责再次串扰
- `TASK_009` 已冻结 `6.7` 首批同步范围与可见承载方式，当前也已完成 `WatchConnectivitySyncing` 最小实现；但轻量重试仍停留在边界约束层，尚未补到更强的 transport 级容错与恢复验证
- App Store 合规项若后置处理，容易阻塞提审
- 帮助与支持页所需的最终 FAQ 内容、支持联系方式和隐私政策正式 URL 尚未冻结；不阻塞 `6.6` 页面骨架实现，但会阻塞提审收口
- watch 震动体验尚未完成真机验证，可能导致震动强度、优先级体感与设计口径存在偏差
- `6.5` 真实动作计数已完成本地最小实现，但 `2026-04-21` 真实 `iPhone 16 + Apple Watch Series 8` 验证不通过：首次校准引导与完成反馈不足、校准采样窗口过短、正常深蹲未能驱动 `live` 计数、正式路径仍暴露“模拟识别”与 `+1 / -1`；当前已通过 `TASK_017` 收敛为必须修复项，修复并重新真机回传前不得宣称“真实可用闭环完成”

### 低风险

- companion / 同步相关 iOS 模拟器测试链路在扩大覆盖面时仍可能卡住；本轮已成功跑通 1 条定向 `xcodebuild test`，但完整 `xcodebuild test` / `test-without-building` 仍未完成更大范围执行，当前影响的是回归覆盖强度，而不是 `6.7` 最小闭环成立

## 应对原则

- 高风险先做最小可验证方案
- 中风险提前建文档和门禁，避免后补
- 风险状态变化后，必须在 checkpoint 前更新本文件
